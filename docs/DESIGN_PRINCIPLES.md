# Fantasy HQ Design Principles

This is a living, versioned product document. Principles may be added, revised,
merged, renumbered, or removed as Fantasy HQ evolves.

## 1. Recommendations over calculations

Fantasy HQ should never display a calculation when it can display a
recommendation.

The live-draft surface leads with the player, action, confidence, and the few
reasons needed to make the pick. Scores and component calculations remain
available through progressive disclosure for users who want to inspect them.

## 2. The draft must remain playable above the fold

On a typical laptop, a manager should see the current recommendation, primary
Draft action, four alternatives, roster state, and current draft position
without scrolling.

## 3. Every player receives the same decision language

Recommended and comparison players use the same information architecture.
Changing the selected player must not remove context or replace conclusions
with a weaker comparison view.

## 4. Planning supports decisions without competing with them

Peek Ahead, safe-to-wait guidance, manager roster state, and the snake board
belong to a secondary Planning layer. They remain accessible between picks but
must not compete visually with the immediate draft decision.
